# CS1 Benchmark Problem Authoring Guidelines

Each benchmark problem shall contain the following components.

1. Benchmark ID
2. Problem Title
3. Learning Outcome
4. Bloom's Taxonomy Level
5. Difficulty Level
6. Programming Construct(s)
7. Problem Type
8. Computational Thinking Skill
9. Problem Statement
10. Input Format
11. Output Format
12. Constraints
13. Sample Input
14. Sample Output
15. Hidden Test Cases
16. Reference Solution
17. Reference Solution Strategy

Every benchmark problem must:

- Be solvable using standard Python 3.
- Require no external libraries.
- Have deterministic output.
- Be independently testable.
- Be appropriate for introductory programming students.
